libAttachSQL

libAttachSQL is a lightweight, non-blocking C API for MySQL servers originally created by HP's Advanced Technology Group. It is desgined to be easy to use and easy to contribute to.

WARNING: THIS PROJECT IS IN BETA, API MAY CHANGE!

Project Information

The libAttachSQL source code is hosted on GitHub. All code is automatically tested using Travis CI. The status of what is currently being worked on can be seen on our Waffle.io project board.

Documentation is in reStructuredText files included in the source and is compiled to generate our Read The Docs documentation.

Questions?

Feel free to file an issue, talk to us on#libAttachSQL on Freenode or email andrew@libattachsql.org

News

Version 0.9.0 RC released!

2014-10-14

The previous beta of libAttachSQL was pretty successful so not a huge amount has changed for the RC. In addition to the standard source releases there are now binary packages! There are currently packages for two Ubuntu versions and two RedHat/CentOS versions. As we progress through RC and GA we will build for more platforms. The major things to note with this release are:

  • 3 new API calls for prepared statements
  • A new connection grouping feature so multiple connections can run from a single event loop
  • Compile fixes
  • Packaging

You can use the following links to download libAttachSQL 0.9.0RC:

  • Source Tarball
  • RHEL 6.x/CentOS 6.x x86_64 library RPM and development library RPM
  • RHEL 7.x/CentOS 7.x x86_64 library RPM, development library RPM and debuginfo RPM
  • PPA for Ubuntu 12.04 (Precise) and 14.04 (Trusty)

Version 0.5.0 Beta released!

2014-10-08

After much hammering with the new sysbench driver for libAttachSQL we have a new much more stable beta of libAttachSQL ready. Major changes in this release are:

  • 11 major bug fixes, most of them crashing bugs
  • Lots more documentation and examples
  • More tests
  • A couple of minor features
  • A new semi-blocking mode
  • A more consistent API for error handline

The last item in that list makes this release API incompatible with previous alpha and beta releases of libAttachSQL but makes the API much more consistent than before. This is a source-only release which can either be obtained from GitHub or a tarball can be downloaded here.

Version 0.4.0 Beta released!

2014-09-22

The first beta version of libAttachSQL has been released today. Instead of lots of little features we bring you one huge feature (Server-side Prepared Statement support) along with bug fixes. Here is a breakdown of what to expect with this release:

  • Compiler fixes for RedHat/CentOS 6.x, GCC 4.9 and Python 3.x (Python is used for documentation building)
  • Much faster building for multi-core systems
  • Buffer and SSL bug fixes
  • Prepared Statement Support!!!

This is a source-only release which can either be obtained from GitHub or a tarball can be downloaded here.

Version 0.3.0 Alpha released!

2014-09-01

Today marks the third alpha release of libAttachSQL. We are well on our way to the first stable release, this release adding many new fixes and features such as:

  • Protocol compression support
  • SSL support (using OpenSSL)
  • Enhanced testing (and fixes)

As before the alpha releases are source-only releases. You can either get the source directly from GitHub or a tarball of this release downloaded here.

Version 0.2.0 Alpha released!

2014-08-15

Hot on the tails of the first alpha release we have released the second alpha of libAttachSQL. This includes many fixes and enhancements including:

  • Improved memory handling
  • Buffered result set support
  • Asynchronous connect on first query
  • Documentation improvements including examples

For now this is still a source-only release and the source can be downloaded from GitHub.

Version 0.1.0 Alpha released!

2014-08-08

Our first early release version of libAttachSQL is out. It can currently connect to MySQL and perform basic queries. We do not recommend it for production at this time and the API will likely change a bit as we head towards a beta release.
You can download v0.1.0 directly from github.
